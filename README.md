# 每日三理 · Theoria

每天三個**可應用**的學術理論（人文／社會／經濟／心理／健康…），**同一組講三天、由淺入深**，再加一張**回想卡**複習更早學過的。獨立於 Horizon 的新專案。

> 目前狀態：**靜態雛形**。內容是手寫樣本、排程是示意，還沒接上 AI 生成與排程器。

## 打開來看

直接用瀏覽器開 `index.html`（不需要伺服器）。
- `index.html` — 今日頁：3 個主題（第 1 張可切換三天）＋ 2 張回想卡（點「顯示答案」）。
- `archive.html` — 歷史頁：按領域瀏覽理論庫，標出今天待複習的。

## 兩條軸（核心設計）

1. **深度弧（learning）**：同 3 個理論連續 3 天，**每天都要深**、層層加碼——
   - 第 1 天「入門全貌」：一篇<strong>完整、夠深</strong>的文章（核心概念＋機制＋應用快覽＋侷限），自成一體。
   - 第 2 天「進階與爭論」：更深的機制、子效應家族、**學界爭議與反例**、跨理論連結。
   - 第 3 天「實戰與內化」是整條弧的**收成日**，刻意做得最厚最廣，建議固定六塊：① 跨領域應用地圖（多生活/工作領域 × 戰術 × 例）② 自我防身解法 ③ 與其他理論的組合拳 ④ 反模式（常見錯用）⑤ 可操作的收成清單／SOP ⑥ 本週實作（觀察→介入→建立習慣 三級）。
   - 不是把基本說明切三薄片，而是第 1 天就到位、第 2、3 天往更深推。
   - **每天、每一個分段都要配 1～3 個具體例子**（例子是核心學習因子，不是裝飾）。生成器需把「每段附例」當硬性規格。
   - **文風要詳細、完整、清楚，把每句話講透**——用完整解釋的段落，不要寫成一句一個重點的精簡筆記／電報體。每個概念都要說明「是什麼、為什麼、所以呢」，讓讀者不需自行補完。

2. **間隔複習（memory）**：每天的回想卡與當日主題無關，專門用**主動回想**（先問後答）叫回**更早**做完的理論。一則跑完 3 天弧後進複習佇列，之後在遞增間隔（預設 +7 / +21 / +60 天）被抽回，間隔遞增會自動控制每日複習量。

## 資料模型

`data/theories.json` 是單一事實來源，網站從它 render，排程器更新它：
- `meta.cycle` — 本輪是哪 3 個、第幾天。
- `review_policy.intervals_days` — 複習間隔（可調）。
- 每則 `theory` 帶 `arc.day1/2/3`、`status`（pool→active→completed）、`review.due_dates / next_due`。

## 接下來要做（讓它真的會跑）

1. **母池**：先建約 100–150 個高應用價值理論（按領域分類），每天從池子抽，保證品質＋天然去重。
2. **生成器**（`generate.py`，Gemini）：給定當日要寫的理論 + 是第幾天，產出該天的 arc 內容；正確性停在概念層級、不杜撰具體研究數字，並加一道輕量自查。
3. **排程器**：推進本輪 day 1→2→3；弧完成 → 排入複習；每天挑「到期」的理論出回想卡；全部寫回 `theories.json`。
4. **部署**：沿用 Horizon 那套 —— GitHub Actions 每日 cron（建議外部觸發確保準時）+ GitHub Pages。
5. （可選）每日 Email 提醒，正本仍在網站。

## 待你定的調整點

- 命名（「每日三理 / Theoria」是暫名）。
- 每天「新主題」要 3 個還是更少（少一點＝記更牢）。
- 複習間隔與每日張數（預設 +7/+21/+60、每天 2 張）。
- 語言：純中文，還是中英雙語（沿用 Horizon 的雙語慣例）。
